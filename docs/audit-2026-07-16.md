## Топ-приоритеты (сквозные, по убыванию важности)

1. **Факты ивента (UX, критично).** На сайте нет: точных дат (только «July 2026»), стоимости участия, состава жюри (упоминается 4 раза, но ни одного имени), организатора и контактов. Добавить «key facts»-полосу + секцию жюри. Без этого намерение зарегистрироваться не переживает контакта с реальностью.
2. **Инверсия hero CTA на AI-скине — не выполнена с прошлого аудита.** Вместо инверсии CTA удалили из hero совсем (#61). Сейчас на первом экране нет доминирующего действия: только маленькая «play» и плавающая пилюля регистрации (самый мелкий интерактив на экране, ~29px на мобайле). Вернуть один явный primary «Enter the tournament», «play» — во вторичные. На classic-скине инверсия сделана правильно.
3. **Синхронизировать формат ивента между скинами.** AI-скин: «Two days, Round 1 → Round 2 (invite) → Final». Classic: «45 минут онлайн → оценка → 2-часовой финал», без Round 2. Выбрать канон и рассказать одинаково.
4. **Включить DS-гейт в сборку (1 строка).** `lint:ds` нигде не запускается автоматически — ни CI, ни prebuild, ни husky. Добавить в `package.json`: `"prebuild": "npm run lint:ds && npm run typecheck"` — Vercel выполнит на каждом деплое.
5. **Живой баг кэша (1 строка).** В `vercel.json` правило для `arcade-machine.glb` стоит **до** общего `/assets/(.*)` — на Vercel побеждает последнее совпавшее, поэтому GLB (589 КБ) отдаётся с `max-age=604800` вместо immutable и перекачивается каждую неделю. Переставить блок GLB после `/assets/(.*)`. Проверено на проде.
6. **Универсальная пауза анимации (WCAG 2.2.2, уровень A).** Единственный подтверждённый класс A-провалов: marquee/тикеры (5 компонентов) и autoplay-видео classic-hero без доступной паузы (hover-пауза недоступна с клавиатуры/тача). Один общий тумблер «pause motion» рядом с SoundToggle закрывает всё разом.
7. **Мобильный проход.** Пилюля Apply перекрывает H1 на classic-hero; плавающий кластер (звук + регистрация + свитчер) — тап-таргеты 29–36px при норме 44px и постоянные коллизии с контентом; тач-версия игры работает, но вход на неё с мобайла скрыт (`display:none`), а тайтл-экран показывает клавиатурные подсказки на таче.

## 1. Безопасность — A

Один из самых защищаемых вариантов архитектуры: чистый статический экспорт, ноль серверной поверхности (проверено: нет route-handlers, `'use server'`, middleware), ноль секретов в репозитории **и во всей git-истории**, ноль внешних origin'ов (нет CDN/аналитики/фетчей), CSP замкнут на `'self'`.

Проверено и чисто:
- Единственный `dangerouslySetInnerHTML` — статический JSON-LD с экранированием (`EventJsonLd.jsx:38`), образцовый паттерн.
- `Leaderboard.jsx:50-65` — образцово-защитный парсинг attacker-editable localStorage (try/catch, типы, regex, clamp).
- Внешние ссылки с `rel="noopener noreferrer"`, postMessage-листенеров нет, open-redirect-путей нет.

Рекомендации (не блокеры):
- **CSP:** дописать `; form-action 'self'; frame-src 'none'` в `vercel.json` — единственная директива, которая ещё имеет значение при неизбежном `'unsafe-inline'` (он инхерентен статическому Next-экспорту — нонсы требуют сервера).
- **PII в localStorage:** demo-регистрация пишет `{name, email}` бессрочно (`src/lib/register.js:76-86`), при этом запись никогда не читается обратно. Перейти на sessionStorage или чистить ключ после success-экрана.
- **Next 14.2.35:** все published-CVE живут в серверных код-путях, которые при `output: 'export'` не выполняются. Риск латентный — активируется в день, когда кто-то уберёт статический экспорт ради бэкенда. Апгрейд на Next 15/16 запланировать до этого момента.
- Записать провенанс Draco-декодера (версия + sha256 трёх файлов) в `MODEL-CREDITS.txt`.

## 2. Производительность — A−

Цифры: лендинг `/` — **115 КБ gz** First Load JS; `/classic` — 99 КБ; TTFB прода 147 мс; сборка проходит (17/17 страниц).

Что сделано правильно (проверено по чанкам и живому HTML):
- **Phaser (316 КБ gz) и three.js (167 КБ) полностью отрезаны от лендинга**: Phaser — `await import()` только при открытии игры; 3D-кабинет — `next/dynamic` за гейтом «десктоп ≥1024 + WebGL + не reduced-motion + IntersectionObserver за 600px до финала». Телефоны это вообще не качают.
- LCP: `hero-bg.webp` с `fetchPriority="high"` + emitted preload — подтверждено на проде.
- Шрифты: 3 самохостнутых WOFF2, `font-display: swap`, preload, immutable — дубликат next/font уже убран после прошлого аудита.
- Runtime дисциплинирован: ноль активных ScrollTrigger'ов, 3 passive-scroll-листенера, marquee гасит rAF вне вьюпорта, одометр IO-armed с корректным no-gsap фолбэком.

Найдено:
- **Баг заголовков кэша GLB** — см. топ-приоритет №5 (единственный живой дефект).
- **`classic/floor/*.jpg` — 2206px при рендере ≤900px, поверх `grayscale(1)`+blend**: полный цветовой chroma оплачивается и выбрасывается. Переэкспорт 5 файлов в ~1300px pre-graded WebP (с новыми именами — конвенция кэш-бастинга): 1140 КБ → ~300 КБ, **экономия ~800 КБ** на скролле `/classic`.
- **Мёртвый вес:** `assets/demo/judges/*` + `organizers/*` (1156 КБ) используются только секциями, которые нигде не монтируются; `HeroDisplay3D.jsx` — мёртвый код (не импортируется никем); **15,5 МБ скриншотов закоммичено в корень репо** (в деплой не попадают, но раздувают каждый clone/CI).
- Мелочь: GT-Pressura-Mono 63,7 КБ выглядит несабсеченным (~30 КБ экономии на критическом пути).

## 3. Дизайн-система — B+

Вердикт по рычагу прошлого аудита («enforce the design system»): **выполнен по существу.** От базы 12.07 (176 деклараций font-size на ~30 сырых значениях, hex в ~40 файлах) к: единый документированный леджер (`src/styles/index.css`, 457 строк), **84% типографики на токенах** (209 из 250 деклараций), **ноль сырых hex в CSS вне леджера** (проверено grep'ом, не только линтером), чистое скоуп-разделение скинов (`--c-*` → алиасы на `--k-*`, без второй палитры), `game/palette.js` синхронизирован со всеми 14 значениями посимвольно. Дисциплина disable-комментариев (69 штук, каждый с `-- deliberate: <причина>`, line-scoped, `reportNeedlessDisables`) — выше среднего по индустрии.

Правило «classic-жёлтый — только заливка» — **соблюдено** (ноль `color: var(--c-yellow)`; жёлтый текст только на чёрном с комментарием-обоснованием).

Дыры энфорсмента (по важности):
1. **Гейт не привязан к CI/сборке** — см. топ-приоритет №4.
2. **Цветовые обходные каналы:** `rgb()/rgba()/hsl()` не гейтятся (3 просочились в `crt.css:67-69`); 7 сырых hex в canvas/SVG/JSX (`Cabinet3D.jsx`, `HeroDisplay3D.jsx:158`, `Fighters.jsx:90`) — им место в `palette.js`. Добавить `function-disallowed-list` в stylelint.

4. **champion-card скопирована 5 раз** вместо общего класса: `ClassicChampionFor.css:73-117` (оригинал), `ClassicBattle.css:55-81`, `ClassicArenas.css:44-58`, `ClassicTracks.css:30-60`, `ClassicHow.css:130-168` — ~40 дублированных деклараций, идентичная пара disable «16px/1.2» повторена 8 раз. Извлечь `.c-card` + модификаторы и заминтить `--k-classic-text-card: 16px` / `--k-classic-lh-card: 1.2` — минус ~8 disable и ~40 строк.

## 4. Доступность — B+

Сильнейшая зона — **prefers-reduced-motion решён архитектурно** (три слоя: общий JS-хук во всех драйверах, глобальный CSS kill-switch, per-component media-блоки; даже WebGL-шейдер получает `uMotion: 0`). Диалоги (`RegisterModal`, `GameTakeover`) — образцовые: трапы, Esc, возврат фокуса, aria-описания ошибок. Skip-links на обоих скинах. Классические pinned-сцены — нативный sticky, скролл-хайджека нет. Контрасты токенов проверены и проходят, включая dimmed-white ярусы футера.

Провалы:
- **High (уровень A, WCAG 2.2.2):** 5 компонентов автодвижущегося текста без универсальной паузы (`Marquee.jsx:114-123` — пауза только mouseenter; `ClassicFinalCta.css:28`, `Champion.css:76`, `FinalCta.css:59,71` — паузы нет вовсе) + autoplay-loop видео classic-hero (`ClassicHeroMedia.jsx:35-46`). Решение — один общий «pause motion»-тумблер (см. топ-приоритет №6).
- **Medium:** свёрнутые ответы classic-FAQ остаются в accessibility-tree (`ClassicFaq.css:91-115` — только `0fr`+`overflow:hidden`; добавить `visibility: hidden` с transition-delay); невидимый фокус на активной пилюле ModeSwitcher (золото-на-золоте ~1:1, `ModeSwitcher.css:80-83`); ScaleCanvas математически гасит браузерный зум на широких экранах (dead band до ~250% на ≥2048px); `aria-pressed` + меняющийся лейбл SoundToggle — противоречивая семантика; тап-таргеты футера classic <24px (WCAG 2.5.8).
- **Low:** секция Champion без заголовка; generic-alt'ы у floor-фото; чекбокс согласия без `aria-required`; двойной анонс бренда в AI-hero.

Требует ручного AT-тестирования: Phaser-канвас скринридером, CRT-фликер через PEAT, 320px/400%-зум проход, forced-colors.

## 5. UX — B− (живой прогон прода)

- **5-секундный тест:** AI-скин (дефолтный!) — частичный провал: читается как «ретро-игровой сайт», а не «профессиональное соревнование»; слово *marketing* выпало из визуальной иерархии («ai-kombat»), WHAT/WHO — 13px-моно в углу поверх пиксель-арта. Classic — проходит, лучший экран сайта.
- **CTA:** см. топ-приоритеты №2. Плюс: четыре имени одного действия (`registration` / `register` / `Apply` / `Registration`) — унифицировать; ASCII-обвес `>>> registration <<<` читается как терминальная декорация, а не кнопка; на финальном экране плавающая пилюля дублирует соседний CTA секции.
- **Нарратив:** призы повторены трижды (Champion gets / Prize Vault / FAQ); нумерация стадий начинается со «Stage 03» — 01/02 не существуют, метафора ломает собственный счёт; **на AI-скине ноль реальных доказательств** (ни фото, ни имён, ни лого) — у classic есть «INSIDE THE ARENA»; portировать.
- **Свитчер скинов:** обнаруживаемость хорошая, но «AI MODE / Classic / MCP» не самоописывается как переключатель тем, MCP — вообще другой класс артефакта внутри mode-свитча; выбор скина **не персистится** (вернувшийся посетитель снова получает AI); пилюля постоянно перекрывает контент на мобайле. Бонус-баг: MCP-промпт обещает секцию «judges», которой нет ни на одном скине.
- **Навигация:** на мобайле секционной навигации нет вообще (AI-«level select» появляется только на десктопе при скролле вверх и всегда `aria-hidden` + `tabindex=-1` — клавиатурные пользователи не доберутся до неё никогда); classic-мобайл сводит нав-карту к одной «Apply». Страница ~8100px без способа прыгнуть к FAQ или регистрации.
- **Игра:** с classic-скина входа в игру нет совсем; с мобайла вход скрыт при работающем тач-управлении; выход из игры бросает на верх hero без моста к регистрации. Держать как secondary/easter egg, не как главное действие hero.
- **Копи-баги:** AI-мобайл — задвоение «who it's for **for** senior marketers»; кнопки регистрации подрезаются HUD-полосой здоровья при скролле; mission-текст арен на десктопе клампится в 2 строки без раскрытия (на мобайле виден целиком).

Что хорошо: FAQ (8 острых вопросов с честными ответами), микрокопия финального CTA («5 MIN TO APPLY · 3 QUESTIONS · AI REVIEW · 48 HR RESPONSE») — лучший конверсионный текст сайта, скролл-композиции classic помогают сканированию и чисто деградируют на мобайле, горизонтального оверфлоу нет нигде.

## 6. Качество кода — B−

Скоуп: ~40 коммитов classic-скина (#26–#64), включая диффы 10 самых крупных. Вердикт: **approve-with-changes**. Дисциплина по существу хорошая — SSR/hydration-гарды, reduced-motion, фокус-трапы, GSAP-твины и ScrollTrigger'ы корректно убиваются при анмаунте везде, где реально используются; чистки мёртвых CTA (#61/#62) убрали JSX+CSS+импорты без остатков. Долг сконцентрирован в одном: **логика «compose-on-scroll runway» скопирована в 5 файлов и уже начала дрейфовать.**

- **High — рассинхрон ClassicMenu с ScaleCanvas на тач-планшетах (1024–1439px).** `ClassicMenu.jsx:19-23` считает scale по `w >= 1024 ? w/1440 : 1`, а `ScaleCanvas.jsx:61,64` + `ScaleCanvas.css:31` дополнительно переводят любое `(pointer: coarse)`-устройство в fluid-мобильную раскладку (комментарий прямо называет iPad landscape/Surface). ClassicMenu `pointer: coarse` не проверяет → на 1280px-тач-планшете тело страницы уходит в мобильный reflow, а фиксированная оранжевая нав-карта остаётся ужатой (~0.89×) десктопной с hover-only-аффордансами — ровно тот кейс, ради которого coarse-ветку и писали. Фикс: общий хук `useCanvasScale()` (scale + fluid + coarse-проверка) для обоих компонентов. *Вывод из статического анализа — стоит 30-секундная ручная проверка на 1280×800 с тач-эмуляцией.*
- **Medium — stale closure на coarse-pointer в ScaleCanvas** (`ScaleCanvas.jsx:61`): `matchMedia('(pointer: coarse)')` читается один раз на маунте и замыкается в `measure()`; 2-в-1 устройство при переключении планшет↔ноутбук не подхватит смену без перезагрузки. Паттерн-образец уже есть в кодбазе — `Cabinet3DGate.jsx:50-54` слушает `change`. Перенести чтение внутрь `measure()` или добавить листенер.
- **Medium — мёртвая константа `STICKY_TOP = 200`** (`ClassicBattle.jsx:22`, `ClassicHow.jsx:28`): выглядит несущей, но нигде не используется — реальный оффсет живёт магическим числом в CSS (`top: 200px` с комментарием «== STICKY_TOP»). ESLint это не ловит (проверено). Следующий, кто поменяет константу под новую высоту меню, молча не изменит ничего. Удалить или реально пробросить через `--pin-top`.
- **Medium — путаница имён вокруг `#c-arenas`:** футер называет его «Battle arenas» (`ClassicFooter.jsx:23`), меню — «Case tracks» (`ClassicMenu.jsx:44-46`), на странице заголовок «Case tracks» при `aria-label="Battle arenas"`; при этом компонент `ClassicTracks.jsx` («Tracks» в обеих навигациях) на странице слова «tracks» не содержит вовсе («Choose your fighting style»). Выбрать один лейбл и привести имя файла/aria/заголовок/оба меню к согласию.
- **Medium — устаревший doc-комментарий** в `ClassicChampionFor.jsx:21-23`: описывает `sticky top:60`, тогда как CSS пинит на `top: 270px` (поднято намеренно под меню). Рендер-бага нет (математика runway не зависит от top), но комментарий врёт — классический дрейф рукой скопированного паттерна.
- **Medium — `HeroDisplay3D.jsx` (617 строк) + CSS подтверждён мёртвым:** никто не импортирует, `HeroStage` монтирует `Hero` напрямую. Удалить.
- **Low:** проп `centerPlay` у Hero вырожден (единственный вызов всегда `true` после #61); `1023px` захардкожен в `ScaleCanvas.css:31` при пропе `breakpoint = 1024` (ручная синхронизация); `Cabinet3D.jsx:484-531` дублирует WebGL-проверку и fallback-разметку из `Cabinet3DGate` (недостижимая ветка — свернуть в `useWebglSupport()` + `<CabinetFallback/>`).

**Топ-рефакторинг по цене/пользе:** извлечь `useScrollCompose({...})` — ~150 строк почти дословно скопированной runway-математики (`mqMobile` → `scale=width/1440` → riseTop/travel/lane → runway-высота → отрицательный marginBottom → ResizeObserver + resize + cleanup) в `ClassicChampionFor.jsx:36-84`, `ClassicBattle.jsx:52-95`, `ClassicHow.jsx:58-111`, плюс 4-я и 5-я копии в `ClassicFooter.jsx:36-58` и `ClassicArenas.jsx:53-115`. Именно этот код уже дрейфанул один раз (см. комментарий top:60). Консолидация закрывает большинство Medium-находок разом — и обязательна до того, как 6-я секция скопирует паттерн снова.

Отдельно: `RoundMoments.jsx` (единственный второй пользователь ScrollTrigger) сейчас размонтирован намеренно; cleanup выглядит корректным — перепроверить при включении.
