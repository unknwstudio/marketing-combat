Вердикт

Инженерия — реально сильная: тяжёлая переработка hero «батарейка = PLAY» (PR #93–99) выехала без структурных регрессий, утечек слушателей и нового скролл-оверхеда. Потолок B+ держат две те же гири, что и 16.07 — контент и «ложная безопасность» гейтов.

▲ Что тянет вверх

  • Безопасность A−. Замкнутый same-origin CSP, HSTS+preload, ноль сетевых вызовов, PII только в sessionStorage. Ни один инъекционный вектор не выжил.
  • Перфоманс A−. Phaser, three/R3F и GSAP подтверждённо вне first-load; home = 116 kB JS для 3D+GSAP-страницы.
  • Игра A−. /play бутится с нулём ошибок консоли; поток батарейка→VS→takeover чист; reduced-motion уважается.
  • Реестр дизайн-системы — один аудируемый token-файл на оба скина, зеркалится в palette.js для Phaser.

▼ Что тянет вниз

  • UX/контент C+. Нет точной даты, цены и контакта; два скина противоречат друг другу по формату ивента.
  • Сборочный гейт B. DS-lint и typecheck считаются включёнными, но инертны в проде.
  • «Enforced» ≠ enforced. Гейт обойдён на Vercel, typecheck no-op, «универсальный» motion-тумблер не глушит CRT.
  • Швы переработки. Осиротевший компонент, coarse-pointer breakpoint, два тумблера без focus/ARIA.

Топ-приоритеты

Дедуплицировано по направлениям, отсортировано по важности. Стрип слева = severity, чип справа = объём.

P1Формат ивента противоречит сам себе между скинамиHighСредне

Единственная high-находка, всплывает в 4 направлениях. AI-скин продаёт 3-фазную воронку с инвайт-онли «ROUND 2», classic — «45 мин онлайн → 2 ч в Барселоне» без Round 2; Hero.jsx:41 «Two days» vs classic 45мин+2ч. Внутри classic FAQ и Jury ссылаются на несуществующий «main tour». Код это признаёт: EventJsonLd.jsx:13.

Фикс Один канон структуры → в site.js (как уже сделано с EVENT_DESCRIPTION) → рендерить оба скина + FAQ + Jury + schema из него. Открыто с 16.07.
P2Сборочные гейты обходятся в реальном деплоеMedium1 строка

vercel.json:4 задаёт buildCommand:"next build" → обходит npm-хук prebuild:lint:ds: stylelint/check-css-vars/hover-guard не запускаются ни на одном деплое Vercel, и CI нет. Отдельно: tsconfig.json отсутствует → tsc --noEmit ничего не проверяет (перепроверено вручную).

Фикс buildCommand → npm run build; добавить минимальный tsconfig.json или убрать мёртвый скрипт. Это не отменённое «добавьте гейт» — гейт есть, но не бежит где важно.
P3Нет ключевых фактов ивентаMediumМелко · контент

Единственная дата — «JULY 2026» без числа (Hero.jsx:58, startDate:'2026-07'); цена/«free» нигде; нет ни одного контакта (в legal.js до сих пор литеральные [EMAIL]). Модалка бесфактовая в момент коммита.

Фикс Дешёвое выехать сейчас: строка «Free to apply», реальный email, заполнить [EMAIL]. Точная дата — за контентом. Открыто с 16.07.
P4«Универсальный» тумблер motion не универсален (WCAG 2.2.2)MediumМелко

Сквозные темы

1 «Enforced» ≠ enforced

DS-гейт, typecheck и «универсальный» motion — все присутствуют и считаются активными, но инертны там, где важно. Ложный сигнал безопасности.

2 Контент отстаёт от инженерии

Каждая high-находка — проблема факта/копирайта, не кода. Машина собрана хорошо; сообщение не устаканено — с 16.07 без изменений.

3 Гейт слеп вне CSS

Дрейф там, где stylelint не видит: ~30 сырых цветов в Phaser, копипаст-стили тумблеров, off-ladder px в пилюлях.

4 Швы, а не поломки

PR #93–99 структурно здоровы, но насыпали edge-мусор: осиротевший компонент, устаревшие комментарии, breakpoint-mismatch, 2 тумблера без focus/ARIA.

5 Два скина, одна дрейфующая правда

Формат, нейминг («battle arenas» vs «case tracks»), OG-теги и JSON-LD расходятся между / и /classic, потому что нет единого источника. Централизация в site.js закрывает P1, P7 и nit-нейминг разом.
