## Топ-приоритеты (сквозные, по убыванию важности)

### 🟠 P1 — Формат ивента противоречит сам себе между скинами (и внутри classic). *Средний объём.*
Единственная high-находка, и она всплывает в **четырёх** направлениях. AI-скин ([Stages.jsx:9-32](../src/sections/Stages/Stages.jsx#L9-L32)) продаёт трёхфазную воронку с инвайт-онли «ROUND 2 · LIVE MAIN TOUR»; classic ([ClassicHow.jsx:38-54](../src/sections/ClassicHow/ClassicHow.jsx#L38-L54)) — двухэтапную «45 мин онлайн → 2 ч в Барселоне» **без Round 2**. [Hero.jsx:41](../src/sections/Hero/Hero.jsx#L41) говорит «Two days», classic — 45мин+2ч. Внутри classic тоже рассинхрон: [ClassicFaq.jsx:23](../src/sections/ClassicFaq/ClassicFaq.jsx#L23) и [ClassicJury.jsx:56](../src/sections/ClassicJury/ClassicJury.jsx#L56) ссылаются на «main tour», которого ClassicHow не описывает. Сам код это признаёт — комментарий в [EventJsonLd.jsx:13](../src/components/EventJsonLd/EventJsonLd.jsx#L13): *«"two days" vs "45 min + 2 hours" copy still disagrees»*. Это же отравляет meta-description и JSON-LD ([site.js:9](../src/lib/site.js#L9) «Two days»).
**Фикс:** выбрать один канон структуры/длительности, положить в единый источник (расширить [site.js](../src/lib/site.js) так же, как уже сделано с `EVENT_DESCRIPTION`), рендерить оба скина + FAQ + Jury + schema из него. Открыто с 16.07 (пункт b).

### 🟡 P2 — Сборочные гейты обходятся в реальном деплое. *1 строка + мелочь.*
Владелец считает DS-гейт включённым, но [vercel.json:4](../vercel.json#L4) задаёт `buildCommand: "next build"`, что **обходит** npm-хук `prebuild: lint:ds` ([package.json:11](../package.json#L11)) — stylelint / check-css-vars / hover-guard **не выполняются ни на одном деплое Vercel**, и нет `.github/` CI, чтобы поймать. Отдельно: `npm run typecheck` — фальшивый гейт: `tsconfig.json` нет, поэтому `tsc --noEmit` печатает справку и ничего не проверяет (проверено вручную: печатает баннер версии, компилирует 0 файлов).
**Фикс:** сменить buildCommand на `npm run build` (1 строка), чтобы хук сработал; добавить минимальный `tsconfig.json` **или** убрать мёртвый скрипт. *(Это НЕ отменённое владельцем «добавьте DS-гейт» — гейт существует; находка в том, что он не запускается там, где важно.)*

### 🟡 P3 — Нет ключевых фактов ивента. *Мелко (упирается в контент).*
Открыто с 16.07 (пункт a). Единственная дата — «JULY 2026» без числа ([Hero.jsx:58](../src/sections/Hero/Hero.jsx#L58), [EventJsonLd.jsx:20](../src/components/EventJsonLd/EventJsonLd.jsx#L20) `startDate:'2026-07'`); цена/«free» нигде не заявлена; **нет ни одного контакта** ([Footer.jsx:34-38](../src/sections/Footer/Footer.jsx#L34-L38) = 3 юр-ссылки; в [legal.js](../src/lib/legal.js) до сих пор литеральные `[EMAIL]`-плейсхолдеры). Модалка регистрации бесфактовая в момент коммита ([register.js:67-68](../src/lib/register.js#L67-L68)).
**Фикс:** дешёвые части бинарны и должны выехать сейчас — строка «Free to apply», реальный контакт-email, заполнить `[EMAIL]`. Точный формат/дата — за контентом.

### 🟡 P4 — «Универсальный» тумблер motion не универсален (WCAG 2.2.2). *Мелко.*
[page.jsx:122](../src/app/page.jsx#L122) рекламирует MotionToggle как механизм 2.2.2, но общесайтовый CRT-flicker/8-сек roll ([crt.css:47,88](../src/effects/crt/crt.css#L47)) и idle-AttractMode ([AttractMode.jsx:53](../src/components/AttractMode/AttractMode.jsx#L53)) слушают **только** ОС-`prefers-reduced-motion`, а не класс `.motion-paused`, который ставит тумблер. Клавиатурный/тач-юзер без ОС-настройки жмёт MOTION OFF и всё равно получает ~6 вспышек/сек, катящуюся полосу и attract-захват. **Фикс:** добавить `.motion-paused .crt--flicker .crt__scanlines, .motion-paused .crt__roll { animation: none }` в crt.css и early-return `isMotionPaused()` в AttractMode/AttractTitle.

### 🟡 P5 — Idle attract-mode крадёт первый клик + прозрачный HUD-бар. *Мелко (проверено вживую).*
Два поведенческих дефекта, воспроизведены в браузере: после ~15с простоя перехватывающий указатель (но `aria-hidden`) оверлей `.attract` на `z-index:2600` **съедает первый клик** пользователя по реальным контролам (регистрация, свитчер); и закреплённый P1/CPU-HUD (`.sh--mid`, `background: rgba(0,0,0,0)`) даёт тексту секций сталкиваться с показаниями на скролле (воспроизведено на scrollY≈3400). **Фикс:** attract — раз в сессию (или пропускать клик-дисмисс сквозь), HUD — непрозрачная/блюр-подложка.

### 🟡 P6 — Слом hero на coarse-pointer 1024–1439px. *Мелко (регрессия переработки).*
Батарейка/тумблеры/PlayFab переключаются в мобильный режим по `(pointer:coarse)`, а само тело hero гейтит только по ширине ([Hero.css:403](../src/sections/Hero/Hero.css#L403)). На iPad Pro landscape (1366px) hero 1440px переполняется, `.hero__playsmall` остаётся видимым **рядом** с уже-`/play` батарейкой (дубль PLAY), а верхне-правые тумблеры отрываются от батарейки. **Фикс:** свести все четыре на одну ветку `(max-width:1023px),(pointer:coarse)`.

### 🟡 P7 — Пороутовые метаданные + структурные данные. *Мелко.*
[layout.jsx:31-45](../src/app/layout.jsx#L31-L45) хардкодит `openGraph.url/title/description`, поэтому `/classic` и `/mcp` отдают **домашний** URL и тайтл в OG/Twitter (проверено: `out/classic.html` → `og:url = https://marketing-combat.vercel.app`). И `/classic` — индексируемая, в sitemap, event-страница — **не отдаёт JSON-LD** вовсе (проверено: `out/index.html` = 1 блок, `out/classic.html` = 0), хотя JSDoc в [EventJsonLd.jsx:4-6](../src/components/EventJsonLd/EventJsonLd.jsx#L4-L6) утверждает обратное. **Фикс:** пороутовый `openGraph`-override + смонтировать `<EventJsonLd/>` в [ClassicApp](../src/app/classic/ClassicApp.jsx).

### 🟡 P8 — Ни одного названного организатора/жюри на живом сайте. *Мелко.*
Реальный ростер существует как **мёртвые данные** ([ClassicOrganizers.jsx:11-16](../src/sections/ClassicOrganizers/ClassicOrganizers.jsx#L11-L16): Fedoseev / Kumar Vyas / Sholina), но оба скина его закомментили. ## Сквозные темы

1. **«Enforced» на деле не enforced.** DS-гейт, typecheck-скрипт и «универсальный» motion-тумблер — все *присутствуют и считаются активными*, но *инертны там, где важно*: гейт обойдён на Vercel, typecheck no-op, тумблер не подключён к CRT/AttractMode. Паттерн — **ложный сигнал безопасности**.
2. **Контент отстаёт от инженерии.** Каждая находка high — это проблема *факта/копирайта* (противоречие формата, нет даты/цены, спрятанные организаторы), а не кода. Машина собрана хорошо; сообщение, которое она несёт, не устаканено — и это не изменилось с 16.07.
3. **Гейт слеп вне CSS.** Дрейф концентрируется ровно там, где stylelint не видит: ~30 сырых цветов в Phaser-слое, вручную синхронизируемые копипаст-стили тумблеров, off-ladder `14px`/`32px` в переработанных пилюлях.
4. **Переработка оставила швы, а не поломки.** PR #93–99 структурно здоровы, но насыпали edge-мусор: осиротевший компонент, устаревшие комментарии, coarse-pointer breakpoint-mismatch, два тумблера без focus/ARIA-обработки, которая есть у соседей.
5. **Два скина, одна дрейфующая правда.** Формат, нейминг («battle arenas» vs «case tracks», «ROUND 01» vs «ROUND 1»), OG-теги и JSON-LD расходятся между `/` и `/classic`, потому что нет единого источника. Централизация в [site.js](../src/lib/site.js) закроет P1, P7 и nit-нейминг разом.

## Что реально сильно (сбалансированно)

- **Безопасность настоящая и полная** — замкнутый same-origin CSP, HSTS с preload, X-Frame DENY, экранированный JSON-LD, зажатые localStorage-чтения, ноль сетевых вызовов, PII только в sessionStorage и никогда не передаётся, пропатченный Next 14.2.35 (серверная CVE-поверхность мертва при static export). Ни один инъекционный вектор не выжил.
- **Перфоманс-дисциплина образцовая** — Phaser ([FightGame.jsx:43](../src/components/FightGame/FightGame.jsx#L43)), three/R3F (двойной гейт через [Cabinet3DGate](../src/components/Cabinet3D/Cabinet3DGate.jsx)) и GSAP — все подтверждённо *вне* first-load лендинга; home = 116kB First Load JS для 3D+GSAP-страницы; sticky-док чисто на CSS от одной `--hp-h` без единого нового scroll-слушателя.
- **Переработанный hero держится** — батарейка-как-PLAY это настоящий `<a href="/play">`, читаемый как кнопка в покое; icon-only тумблеры сохраняют доступное имя в visually-hidden тексте (проверено вживую); каждый фикс-элемент смонтирован ровно один раз без дублей.
- **Игра и её in-page takeover прочны** — `/play` бутится на 480×270 с нулём ошибок консоли, поток батарейка→VS→GameTakeover работает без навигации/ошибок, teardown чисто отменяет mid-boot, reduced-motion уважается по shake/flash/haptics/attract.
- **Реестр дизайн-системы лучший в классе** — один аудируемый token-файл, сворачивающий оба скина с role-комментариями, зеркалящийся в [palette.js](../src/game/palette.js) для Phaser, stylelint + check-css-vars + hover-guard проходят чисто (локально).
- **Модалка по-настоящему доступна** — `role=dialog`, `aria-modal`, focus-trap, Escape возвращает фокус, scroll-lock, `role=status` success-панель, demo-режим раскрыт.
- **Дерево худое и честно прополото** — все 8 runtime-зависимостей используются, ноль мёртвых импортов на смонтированных страницах, а «12MB скриншотов в корне» — устаревшая тревога: они в `.gitignore`, в clone не попадают.
